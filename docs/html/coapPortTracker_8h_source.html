22/* Port tracking works as follows:
23 *
24 * For CoAP going from local UDP to serial:
25 * - If the code is 0.00 (Empty message), map MID to port and send as-is
26 * - If the token length is more than 5, map MID to port and send as-is (can't add token tracking)
27 * - Otherwise, prepend a marker byte followed by the port (little-endian) to the tokens
28 *
29 * For CoAP going from serial to local UDP:
30 * - If the code is 0.00, or there are fewer than 3 tokens, or the first token is not the marker
31 * - If the MID is in the map, send to that port
32 * - Otherwise, send to default CoAP server port
33 * - Otherwise (message is non-Empty with token merker present), extract port from token and send
34 * - Marker and port tokens are removed from the message after being extracted
35 *
36 * Port/MID mappings can be overwritten by newer messages with the same MID but different port, and
37 * expire after midTImeoutTime.
38 */
